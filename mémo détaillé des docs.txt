//
// Description : Ce fichier est le point d'entrée principal de l'application web.
// Il définit la structure HTML globale du jeu, inclut les feuilles de style CSS,
// les bibliothèques externes (comme break_infinity.min.js), et charge tous
// les fichiers JavaScript modulaires qui contiennent la logique du jeu.
// Il ne contient aucune logique JavaScript active, servant uniquement de conteneur
// pour les éléments DOM et les inclusions de scripts.
//
// Objectif : Structurer l'interface utilisateur et orchestrer le chargement de tous

//
// ------------------ Dépendances (Inclusions de Fichiers) ------------------
//
// - style.css : Fournit les styles visuels pour l'interface du jeu.
//   Impact : Indispensable pour l'apparence générale, les thèmes et les modes jour/nuit.
//
// - break_infinity.min.js : Bibliothèque JavaScript pour la gestion des grands nombres.
//   Impact : Doit être chargé en premier car il expose une variable globale `Decimal`
//             utilisée par tous les autres scripts JavaScript pour les calculs numériques.
//
// - core.js : Fonctions principales du jeu, gestion de l'état global, sauvegarde/chargement,
//             et la boucle de jeu.
//   Impact : Contient la fonction `initializeGame()` appelée au chargement de la page.
//
// - data.js : Données statiques du jeu (coûts, multiplicateurs, définitions des compétences,
//             quêtes, succès, etc.).
//   Impact : Fournit les données utilisées par tous les modules de logique.
//
// - ui.js : Gestion de l'interface utilisateur, mise à jour de l'affichage des ressources,
//           états des boutons, visibilité des sections.
//   Impact : Manipule la visibilité des éléments DOM définis ici (ex: `display: none;`).
//             Interagit avec les éléments de sélection de thème et de mode jour/nuit.
//
// - studies.js : Logique spécifique aux achats et à la production liés aux études
//                (Élèves, Classes, Images, Professeurs).
//   Impact : Référence et met à jour les éléments DOM des sections d'achat d'études.
//
// - automation.js : Logique de l'automatisation des achats.
//   Impact : Référence et met à jour les éléments DOM de la section d'automatisation.
//
// - skills.js : Logique de l'arbre de compétences (achat, réinitialisation, application des effets).
//   Impact : Référence et met à jour les éléments DOM de l'arbre de compétences.
//
// - ascension.js : Logique du mécanisme d'ascension (calcul des PA, achats d'Ascension, réinitialisation douce).
//   Impact : Référence et met à jour les éléments DOM de la section d'ascension.
//
// - prestige.js : Logique du mécanisme de prestige (calcul des PP, achats de Prestige, réinitialisation super douce).
//   Impact : Référence et met à jour les éléments DOM de la section de prestige.
//
// - settings.js : Logique des paramètres du jeu (thème, progression hors ligne, minimisation des ressources).
//   Impact : Interagit directement avec les éléments DOM de la section paramètres,
//             notamment le `<select id="themeSelector">` et les boutons de mode jour/nuit
//             (`dayModeButton`, `nightModeButton`) pour appliquer les classes CSS au `<body>`.
//
// - quests.js : Logique du système de quêtes (définition, suivi de progression, récompenses).
//   Impact : Référence et met à jour les éléments DOM de la section des quêtes.
//
// - achievements.js : Logique du système de succès (définition, vérification des conditions,
//                     récompenses, infobulles).
//   Impact : Référence et met à jour les éléments DOM de la section des succès et de l'infobulle.
//
// - events.js : Gestion centralisée de tous les écouteurs d'événements DOM (clics sur les boutons,
//               changements d'état, etc.).
//   Impact : Attache les gestionnaires d'événements à la plupart des éléments DOM interactifs
//             définis dans ce fichier.
//
// ------------------ Variables Clés (JavaScript) ------------------
//
// Aucune variable JavaScript clé globale n'est définie ou gérée directement dans ce fichier.
// Il sert uniquement de conteneur pour les éléments DOM et les inclusions de scripts.
//
// ------------------ Fonctions Clés (JavaScript) ------------------
//
// Aucune fonction JavaScript n'est définie ou exécutée directement dans ce fichier,

//   - Boutons d'action : `resetSkillsButton`, `buyAllSkillsButton`.
//     Impact : Gérés par `skills.js` et `events.js`.

// ------------------ Notes Spécifiques ------------------
//
// - Ordre des scripts : L'ordre d'inclusion des balises `<script type="module">` est crucial
//   en raison des dépendances entre les modules JavaScript. `break_infinity.min.js` doit être

//   qui garantit que leurs dépendances sont déjà disponibles.
//
// - Classes CSS dynamiques : De nombreux éléments ont des styles `display: none;` ou des classes
//   (comme `active`, `unlocked`, `can-afford`, `cannot-afford`, `normal-theme`, `day-mode`)
//   qui sont manipulés dynamiquement par les scripts JavaScript (principalement `ui.js`, `core.js`, `settings.js`)
//   pour contrôler leur visibilité et leur apparence.
//
// - Bonnes pratiques : Le code HTML est structuré de manière sémantique, avec des IDs clairs
//   pour faciliter la manipulation par JavaScript et des commentaires pour la lisibilité.
//   Les styles inline ont été minimisés au profit de classes CSS gérées par `style.css`.

// ------------------ Fiche Mémo : break_infinity.min.js ----------------------------
//
// Description : Ce fichier est la version minifiée de la bibliothèque JavaScript "Break Infinity".
// Il fournit une implémentation de nombres à virgule flottante de très haute précision,
// capable de gérer des valeurs beaucoup plus grandes et plus petites que les nombres
// natifs de JavaScript (qui sont des nombres à virgule flottante de double précision IEEE 754).
//
// Objectif : Dans les jeux incrémentaux, les quantités de ressources, les coûts et les productions
// peuvent rapidement atteindre des valeurs astronomiques (bien au-delà de 1e308, la limite de JavaScript).
// Cette bibliothèque résout ce problème en permettant de représenter et de manipuler des nombres
// avec une mantisse et un exposant séparés, prévenant ainsi les erreurs d'arrondi et les "Infinity"
// prématurés.
//
// ------------------ Concepts Clés et Fonctionnement ------------------
//
// 1.  **Objet `Decimal`** : La bibliothèque expose une classe globale nommée `Decimal`.
//     Tous les nombres qui sont susceptibles de devenir très grands (ou très petits)
//     doivent être instanciés comme des objets `Decimal` (ex: `new Decimal(1e300)`).
//
// 2.  **Représentation Interne** : Un nombre `Decimal` est généralement stocké sous la forme de :
//     * `mantissa` (mantisse) : Un nombre à virgule flottante standard (généralement entre 1 et 10, ou -1 et -10 pour les négatifs).
//     * `exponent` (exposant) : Un entier représentant la puissance de 10 par laquelle la mantisse est multipliée.
//     Par exemple, 1e309 serait représenté comme `mantissa: 1, exponent: 309`.
//
// 3.  **Opérations Arithmétiques** : La bibliothèque surcharge toutes les opérations arithmétiques
//     standard (`add`, `sub`, `mul`, `div`, `pow`, `log`, `sqrt`, etc.) pour qu'elles fonctionnent
//     avec les objets `Decimal`. Il est crucial d'utiliser ces méthodes de la bibliothèque
//     pour toutes les opérations impliquant des nombres `Decimal`, et non les opérateurs JavaScript natifs.
//     Ex: `nombre.add(autreNombre)` au lieu de `nombre + autreNombre`.
//
// 4.  **Conversion** : Des méthodes sont fournies pour convertir des nombres natifs ou des chaînes
//     en objets `Decimal` (`new Decimal(value)`, `Decimal.fromNumber(value)`, `Decimal.fromString(value)`)
//     et pour reconvertir des objets `Decimal` en nombres natifs ou chaînes pour l'affichage
//     (`toNumber()`, `toString()`, `toExponential()`, `toFixed()`).
//
// 5.  **Précision** : La bibliothèque gère la précision des calculs pour éviter les pertes
//     significatives qui se produiraient avec les nombres natifs de JavaScript.
//
// ------------------ Dépendances ------------------
//
// - Aucune dépendance externe. C'est une bibliothèque autonome.
//
// ------------------ Impact sur d'autres Modules ------------------
//
// - **`core.js`** : Toutes les variables d'état du jeu qui représentent des quantités
//   (Bons Points, Images, Élèves, PA, PP, etc.), des productions (BP/s), des coûts,
//   ou des multiplicateurs qui peuvent devenir très grands (ou très petits) DOIVENT
//   être des instances de `Decimal`. Toutes les opérations sur ces variables doivent
//   utiliser les méthodes de l'objet `Decimal`.
// - **`data.js`** : Toutes les valeurs numériques initiales ou constantes définies dans `data.js`
//   qui sont destinées à être manipulées comme de grands nombres doivent être des objets `Decimal`.
// - **`studies.js`, `ascension.js`, `prestige.js`, `automation.js`, `skills.js`, `quests.js`, `achievements.js`** :
//   Tout module qui effectue des calculs arithmétiques ou des comparaisons avec les variables
//   d'état du jeu (qui sont des `Decimal`) doit impérativement utiliser les méthodes de l'objet `Decimal`
//   pour ces opérations.
// - **`ui.js`** : Lors de l'affichage de ces nombres, `ui.js` doit utiliser les méthodes de
//   formatage de `Decimal` (comme `toFixed()`, `toExponential()`, `toString()`) ou la fonction
//   `formatNumber` de `core.js` qui elle-même utilise `Decimal`.
//
// ------------------ Notes Spécifiques ------------------
//
// - **Version Minifiée** : Le fichier `break_infinity.min.js` est une version compressée
//   pour la production. Son code est difficilement lisible directement.
// - **Chargement Global** : Cette bibliothèque est généralement chargée via une balise `<script>`
//   dans `index.html` AVANT tous les autres scripts du jeu, rendant la classe `Decimal`
//   disponible globalement.
// - **GitHub du Créateur** : Pour des informations détaillées sur l'API et le fonctionnement
//   interne, il est recommandé de consulter le dépôt GitHub officiel de la bibliothèque
//   (par exemple, le dépôt de "Break Infinity" par Patashu).
//
// ---------------------------------------------------------------------

// - `newSettingsUnlocked` (boolean) : Flag pour débloquer de nouveaux paramètres.
// - `statsButtonUnlocked` (boolean) : Flag pour débloquer le bouton des statistiques.
//
// ------------------ Fonctions Clés Définies et Exportées ------------------
//
// - `toggleTheme()` : (maj 30/05 theme & style)
//   Description : Bascule l'état du thème visuel entre le mode jour et le mode nuit.
//   Si l'option de thème n'est pas encore débloquée (`themeOptionUnlocked` est false),
//   elle vérifie si le joueur possède suffisamment d'`images` (coût : 10 Images) pour la débloquer.
//   Une fois débloquée ou si elle l'était déjà, elle inverse la valeur de `isDayTheme`
//   et applique la classe CSS correspondante (`dark-theme` ou non) à l'élément `<body>`.
//   Elle déclenche ensuite une mise à jour de l'interface et une sauvegarde du jeu.
//   Appelée par : `events.js` (via le clic sur le bouton de thème ou le sélecteur).
//   Impacts : Modifie l'apparence globale du jeu via `style.css`.
//
// - `toggleOfflineProgress(isChecked)` :
//   Description : Active ou désactive la fonctionnalité de progression hors ligne.
//   Met à jour la variable d'état `offlineProgressEnabled` en fonction de la valeur
//   `isChecked` (provenant d'une case à cocher). Affiche une notification et sauvegarde le jeu.
//   Appelée par : `events.js` (via le changement d'état de la case à cocher).
//   Impacts : Affecte les calculs de progression du jeu lorsque le joueur est absent.
//
// - `toggleMinimizeResources()` :
//   Description : Bascule l'affichage entre le mode normal des ressources et un mode minimaliste.
//   Inverse la valeur de `minimizeResourcesActive`. Appelle `updateSectionVisibility()`
//   pour gérer l'affichage/masquage du panneau minimaliste et `updateDisplay()` pour rafraîchir
//   les valeurs. Sauvegarde l'état du jeu.
//   Appelée par : `events.js` (via le clic sur le bouton "Minimalist Resources").
//   Impacts : Modifie la disposition et la quantité d'informations affichées sur les ressources.
//
// - `openStats()` :
//   Description : Ouvre la modale des statistiques du jeu. Appelle la fonction
//   `openStatsModal()` de `ui.js` pour gérer l'affichage de la modale.
//   Appelée par : `events.js` (via le clic sur le bouton "Statistiques").
//   Impacts : Affiche un résumé détaillé des progrès du joueur.
//
// ------------------ Éléments DOM Clés (Interagis par settings.js) ------------------
//
// Ce module interagit directement avec l'élément `<body>` pour appliquer les classes de thème.
// Il ne récupère pas directement d'autres éléments DOM par `document.getElementById` ;
// les interactions avec les contrôles de l'interface utilisateur (sélecteur de thème,
// cases à cocher, boutons) sont gérées par `events.js` qui appelle les fonctions de ce module.
//
// - `document.body` : L'élément `<body>` de `index.html`. Les classes `dark-theme`
//   (ou l'absence de celle-ci) sont appliquées ici pour changer l'apparence globale. (maj 30/05 theme & style)
//
// ------------------ Logique Générale ------------------
//
// `settings.js` est une couche de contrôle pour les préférences du joueur. Il ne contient
// pas de logique de jeu complexe (calculs de production, achats d'unités). Au lieu de cela,
// il modifie des variables d'état globales (`core.js`) et interagit avec la couche UI
// (`ui.js`) pour refléter ces changements visuellement. Les appels à `saveGameState()`
// sont cruciaux pour que les préférences du joueur soient persistantes entre les sessions.

 * Description : Ce fichier gère toute la logique et les fonctionnalités liées aux quêtes
